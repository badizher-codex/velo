Privacy Policy

Last updated: April 2026  ·  VELO Browser

TL;DR — The short version

  • ✅ VELO collects zero telemetry — no usage data, no analytics, no crash reports sent anywhere.
  • ✅ All your data (passwords, history, bookmarks) is stored locally on your device only.
  • ✅ No account required. No cloud. No tracking.
  • ✅ VELO is fully open source — you can audit every line of code.

1. Who we are

VELO Browser is an open-source project maintained by contributors listed on GitHub. It is not a company. There is no legal entity collecting your data.

2. Data VELO does NOT collect

VELO does not collect, transmit, or sell any of the following:

  • Browsing history or visited URLs
  • Search queries
  • Passwords or vault contents
  • Installed extensions or settings
  • Device identifiers, IP addresses, or hardware fingerprints
  • Crash reports or diagnostic data
  • Usage statistics or interaction events

3. Data stored locally on your device

The following data is saved only on your computer, inside %AppData%\VELO\, and never leaves your device:

Data Location Encrypted
Browsing historySQLite databaseNo
BookmarksSQLite databaseNo
Saved passwords (Vault)SQLite + AES-256Yes — AES-256
Settings & preferencesJSON fileNo
WebView2 cache & cookiesWebView2 profileNo

4. Network connections VELO makes

VELO makes the following outbound network connections on your behalf:

DNS-over-HTTPS (DoH)

When DoH is enabled, DNS queries are sent to the provider you configure (default: Cloudflare 1.1.1.1 or Quad9). Your chosen provider's privacy policy applies. You can disable DoH or change the provider in Settings → Network.

Blocklist updates

VELO periodically downloads updated tracker blocklists from EasyPrivacy (easylist.to). Only a GET request is made — no personal data is sent.

AI Assistant (VeloAgent)

VeloAgent runs entirely locally using Ollama or LM Studio. No data is sent to any cloud AI service unless you explicitly configure a remote endpoint in Settings → AI. When using a local model, all inference happens on your machine.

WebView2 Runtime (Microsoft)

VELO uses Microsoft Edge WebView2 to render web pages. WebView2 may make its own connections (SmartScreen checks, Edge update pings) governed by Microsoft's privacy policy. You can review WebView2 network settings in the browser settings.

Normal web browsing

When you visit websites, those sites receive your IP address and standard HTTP headers — exactly as with any browser. VELO actively blocks trackers and injects fingerprint noise to reduce your digital footprint.

5. Authenticode code signing

VELO release binaries are signed using SignPath Foundation, a free code-signing service for open-source projects. This signing process does not transmit any user data — it only proves the binary was built from the official source code.

6. Children's privacy

VELO does not knowingly collect any information from anyone, including children under 13. Since no data is collected at all, there is nothing to disclose.

7. Changes to this policy

If this policy changes, the updated version will be committed to the GitHub repository and this page will be updated with a new date. You can review the full history of changes on GitHub.
